# `vercel-react-best-practices`

## Purpose

Guides React and Next.js performance, rendering, bundle, async, server/client, and architecture best practices based on Vercel engineering guidance.

## Use When

- Writing, reviewing, or refactoring React or Next.js code.
- Planning data fetching, server/client boundaries, Suspense, transitions, or bundle optimization.
- Diagnosing rerenders, hydration issues, performance regressions, or async patterns.

## Best Paired Commands

- `/sdd-specify` for React/Next.js design decisions.
- `/sdd-execute` for implementation.
- `/sdd-test` for performance-sensitive verification.
- `/sdd-validate` for conformance review.
